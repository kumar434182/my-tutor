# Enterprise Learning Platform - Complete Deployment Guide

This guide will walk you through deploying the complete Enterprise Learning Platform on Render.com and GitHub Pages, completely free.

## 🎯 Deployment Architecture

- **Frontend:** GitHub Pages (free static hosting)
- **Backend:** Render.com (free Docker hosting with auto-sleep)
- **Database:** Render.com PostgreSQL (free 256MB database)
- **CI/CD:** GitHub Actions (free automated deployments)

**Total Cost: $0.00/month**

---

## 📋 Prerequisites

Before starting, ensure you have:

1. ✅ A GitHub account with the repository already pushed
2. ✅ A Render.com account (create at [https://render.com](https://render.com))
3. ✅ Your GitHub repository URL ready

---

## 🚀 Deployment Steps (In Order)

### Step 1: Deploy Backend & Database on Render.com (10 minutes)

This step will create both your backend service and PostgreSQL database on Render.

1. **Log in to Render Dashboard**
   - Go to [https://dashboard.render.com](https://dashboard.render.com)
   - Sign in with your GitHub account

2. **Create a Blueprint Deployment**
   - Click **New +** → **Blueprint**
   - Select **Public Git repository**
   - Paste your GitHub repository URL
   - Click **Connect**

3. **Review and Apply Blueprint**
   - Render will detect the `render.yaml` file
   - You will see two services:
     - Backend service (Spring Boot Docker container)
     - PostgreSQL database (256 MB free tier)
   - Click **Apply Blueprint**

4. **Configure Environment Variables**
   - You will be prompted to enter values for:
     - `SPRING_DATASOURCE_URL`: Will be auto-filled by Render
     - `SPRING_DATASOURCE_USERNAME`: `postgres`
     - `SPRING_DATASOURCE_PASSWORD`: Generated by Render (or set your own)
     - `JWT_SECRET`: Generate a random string using: `openssl rand -base64 32`
   - Click **Create Services**

5. **Wait for Deployment**
   - Render will build the Docker image and deploy
   - This takes 5-10 minutes
   - You can monitor progress in the dashboard

6. **Get Your Backend URL**
   - Once deployed, Render will show your backend URL
   - It will look like: `https://learning-platform-backend-xyz1.onrender.com`
   - **Save this URL** - you need it for the frontend

---

### Step 2: Update Frontend Configuration (2 minutes)

Now you need to tell your Angular frontend where to find the backend API.

1. **Edit Environment File**
   - Open `frontend/src/environments/environment.prod.ts`
   - Replace the `apiUrl` with your Render backend URL:
   ```typescript
   export const environment = {
     production: true,
     apiUrl: 'https://your-render-backend-url.onrender.com/api/v1'
   };
   ```

2. **Update GitHub Actions Workflow**
   - Open `.github/workflows/deploy-frontend.yml`
   - Update the `BASE_HREF` environment variable:
   ```yaml
   env:
     BASE_HREF: '/your-repository-name/'
   ```
   - If using a custom domain, set `BASE_HREF: '/'`

3. **Commit and Push**
   ```bash
   git add frontend/src/environments/environment.prod.ts
   git add .github/workflows/deploy-frontend.yml
   git commit -m "Configure frontend for Render backend"
   git push origin main
   ```

---

### Step 3: Enable GitHub Pages (2 minutes)

1. **Go to Repository Settings**
   - Navigate to your GitHub repository
   - Go to **Settings** → **Pages**

2. **Configure Pages**
   - Under **Build and deployment**, select **Deploy from a branch**
   - Wait for the GitHub Actions workflow to run (check the **Actions** tab)
   - Once the workflow completes, return to the **Pages** settings

3. **Select Deployment Branch**
   - Under **Branch**, select `gh-pages` and `/ (root)`
   - Click **Save**

4. **Get Your Frontend URL**
   - GitHub will show your site URL
   - It will look like: `https://your-username.github.io/your-repository-name/`
   - This is your live application!

---

### Step 4: Initialize Database Schema (2 minutes)

The database schema should be automatically initialized, but if it's not:

1. **Connect to Your Database**
   - In Render dashboard, go to your PostgreSQL service
   - Click **Connect** and copy the **PSQL Command**

2. **Run SQL Script**
   - Open a terminal and paste the PSQL command
   - Once connected, copy the contents of `docs/database_postgres.sql`
   - Paste it into the PSQL terminal
   - Press Enter to execute

## ✅ Verification

Your platform is now live! Let's verify everything works:

1. **Test Frontend**
   - Open your GitHub Pages URL in a browser
   - You should see the login page

2. **Test Backend API**
   - Open your Render backend URL + `/swagger-ui.html`
   - Example: `https://your-backend.onrender.com/swagger-ui.html`
   - You should see the Swagger API documentation

3. **Test Login**
   - On the frontend login page, try logging in
   - Use any email and password (the system will create a test account)
   - If login works, the frontend-to-backend connection is working!

## ⚠️ Important Notes

### Cold Starts (Auto-Sleep)

Because Render's free tier auto-sleeps after 15 minutes of inactivity:
- **First request after sleep:** 30-60 seconds delay
- **Subsequent requests:** Normal speed
- This is normal and expected behavior

### Database Limits

- **Storage:** 256 MB (sufficient for thousands of courses and users)
- **Connections:** Limited but sufficient for free tier
- **Backups:** Not included in free tier

### GitHub Pages Limits

- **Storage:** 1 GB per repository
- **Bandwidth:** Unlimited
- **Build time:** 10 minutes per deployment

## 🔄 Making Changes

After deployment, whenever you push code to the `main` branch:

1. **Frontend Changes**
   - GitHub Actions automatically builds and deploys to GitHub Pages
   - Changes appear within 2-3 minutes

2. **Backend Changes**
   - You need to manually redeploy on Render
   - Go to Render dashboard → Your service → **Manual Deploy**
   - Or push a new commit and Render will auto-deploy if configured

## 🆘 Troubleshooting

### Frontend Shows 404

- Ensure `BASE_HREF` in the workflow matches your repository name
- Check that GitHub Pages is enabled in repository settings
- Wait for the GitHub Actions workflow to complete

### Backend API Not Responding

- Check that the Render service is running (not sleeping)
- Verify the backend URL in `environment.prod.ts` is correct
- Check Render logs for errors

### Database Connection Errors

- Verify `SPRING_DATASOURCE_URL` is correct
- Ensure database credentials match what you set in Render
- Check that the database is running in Render dashboard
